Privacy Policy

Rask is a self-hosted application. We have no access to your data, your broker, or your usage. This page explains what does and doesn't happen with information when you run Rask.

The short version

✓ Rask stores nothing about you or your usage.

✓ No telemetry, analytics, or tracking of any kind.

✓ No data ever leaves your server — Rask never phones home.

✓ No account, registration, or email address required to use Rask.

What is stored

The only thing Rask writes to disk is the server configuration — your RabbitMQ host, port, and credentials — stored in a .env.local file on the server where you run Rask. This file is not committed to version control (it is listed in .gitignore), is never sent to the browser, and is never accessible from outside your server.

Nothing else is persisted anywhere. All filter states and UI preferences live in memory and reset on page refresh.

Your RabbitMQ data

Queue contents, message payloads, connections, and all other broker data flow between your browser and your own RabbitMQ broker — proxied through the Next.js server you deploy. Agdir Drift AS has no access to any of it, ever.

Rask respects the privacy and terms of the RabbitMQ project. See RabbitMQ's privacy policy and terms and conditions for information about the broker software itself.

Credentials

RabbitMQ credentials are stored server-side in .env.local and are never sent to the browser. API routes read them at request time to authenticate against your broker. They do not appear in any client-facing response, log output, or error message.

Third-party services

The Rask dashboard application you self-host loads no third-party scripts, fonts, or resources from external CDNs at runtime. All assets are served from your own origin.

The public documentation site (rask.agdir.farm) loads fonts from Google Fonts. This involves a standard HTTPS request for font files — no personally identifiable information is shared beyond what is inherent in any HTTP request. The self-hosted Rask application is not affected.

Logs

Rask does not produce application-level logs beyond what Next.js writes to stdout by default (request paths and response codes). What happens to those logs is determined entirely by the environment you deploy Rask in — Agdir Drift AS has no access to them.
