About Rask

An open source RabbitMQ management dashboard by Agdir Drift AS. Named after Ratatoskr, the Norse messenger squirrel.

Why we built it

The native RabbitMQ management UI has not kept pace with how operators actually work. It is functional, but it is also a Relic of 2010 web development — table-heavy, hard to navigate, and impossible to extend. We run RabbitMQ in production and needed something better.

Rask is that something better. It gives you a modern interface, real-time message tracing, topology visualisation, a publishing tool, and a cleaner hierarchy — while still proxying directly to the RabbitMQ Management HTTP API, so every capability is available.

Who we are

Agdir Drift AS is a Norwegian farming-as-a-service company based in Arendal, Norway (Krøgenes Bryggevei 19, 4841 Arendal). We build software that helps farmers manage land, assets, and operations. RabbitMQ is part of our infrastructure, and Rask came out of our own frustration with the tooling. We use it in production ourselves, every day.

Built with

Rask is built on Next.js 16 + React 19, styled with Tailwind CSS v4 and shadcn/ui (Radix UI primitives). Data fetching uses TanStack Query. The topology view uses React Flow. Icons are from Lucide. AMQP operations (publish, peek, purge) run server-side via amqplib. The codebase has unit tests (Vitest) and end-to-end tests (Playwright).

Architecture

All RabbitMQ Management HTTP API calls are proxied through Next.js API routes. The browser never talks to RabbitMQ directly — which simplifies auth, CORS, and lets you run Rask inside a private network without exposing port 15672. 

Browser → Next.js API Routes (/api/rabbitmq/*)
       → RabbitMQ Management API (port 15672)
       → RabbitMQ AMQP (port 5672, server-only)

AMQP operations (peek, publish, purge) are performed server-side. The amqplib library never runs in the browser.

Configuration

All environment variables are optional. Defaults work with a local RabbitMQ instance.

RABBITMQ_HOSTlocalhost
RABBITMQ_MANAGEMENT_PORT15672
RABBITMQ_AMQP_PORT5672
RABBITMQ_USERguest
RABBITMQ_PASSWORDguest
RABBITMQ_VHOST/
